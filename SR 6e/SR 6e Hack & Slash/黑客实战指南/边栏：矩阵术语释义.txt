[size=16pt][b]矩阵术语释义[/b][/size]

[b]基层(Base)：[/b]这是一种用于描述框架式主机底层的替代术语。

[b]底层(Basement)：[/b]框架式主机的操作文件、所有权结构和归档。

[b]光砖(Brick)：[/b]光学堆栈式大型计算机的俚语。

[b]赛博终端(Cyberterminal)：[/b]集赛博碟板和赛博插口功能于一体的台式计算机。这些装置不能随身携带。

[b]数据终端(Dataterm)：[/b]常用的家用电脑、家庭娱乐系统、通讯设备。这些装置不能随身携带。

[b]结构深层(Deep Foundation)：[/b]矩阵本身的结构层，连接着所有的主机结构层。

[b]结构层(Foundation)：[/b]结构层式主机的子结构，它运行着主机的操作代码。这一层位于共鸣域和主机之间。

[b]结构层技术(Foundation-Tech)：[/b]用于创建和维护结构层式主机的技术。

[b]框架(Framework)：[/b]由物理处理器而非结构层生成的主机。这些主机由光学堆栈式大型计算机组成。有时也被称为遗留主机。

[b]混合用户团体(Mixed User Groups，MUGs)：[/b]用于描述已存在的大多数矩阵团体的简称.

[b]新矩阵(New Matrix)：[/b]丹妮尔·德·拉·马尔在2075年对矩阵所做的改动的简称(不是个人所为，而是在她的指挥下)。在这一点上，它不是真正的“新”，但乖戾的老碟客们总是对此耿耿于怀。

[b]新矩阵协议(New Matrix Protocols)：[/b]见新矩阵。

[b]心智层(Noosphere)：[/b]信息传输和存储过程中产生的能量场。这基于通用信息理论。

[b]心智粒子场机制(Nous-Particle Field Mechanics)：[/b]通用信息理论的核心组成部分。这指的是数据和信息是如何通过主机、思想和共鸣域连接起来的。

[b]局域网(LAN)：[/b]局部区域网络。指设备可以从属于小型框架式主机(规模1或2)的地理区域。

[b]光学堆栈式大型计算机(optical stack mainframe)：[/b]用于生成主机的标准大功率计算设备。

[b]个域网(PAN)：[/b]私人区域网络. 这包含了所有与你的主设备相连的设备，该主设备通常是通讯链或赛博插口。

[b]范式(Paradigm)：[/b]这是一个结构层术语，在结构层中包括所有结构层化身和事物在内的环境都被强行匹配某些外观和感觉。这类似于主机的塑模，只是更具侵入性。

[b]光方、光立方和光墙(Quad, Cube, Wall)：[/b]这些是预先组网的4、16或64个光学堆栈式大型计算机。

[b]共鸣域(Resonance Realms)：[/b]几乎不为人知的支撑着结构深层的事物。超链者了解这个。去问他们。

[b]构架(Scaffolding)：[/b]运行主机塑模的系统。这是由结构层中的构架控制节点来操控的。

[b]塑模(Sculpting)：[/b]定义主机用户界面的编程。就像范式一样，只是它不会强迫你穿一些愚蠢的衣服或突然变成一辆车。

[b]壳(Shell)：[/b]在进行任何塑模前的主机。它也是主机，只是有些贫乏。

[b]传输电路(Transfer Circuit)：[/b]企业法庭授权公司用于创建结构层主机壳的代码。这会使得结构层里神奇地长出了一个主机。我相信德·拉·马尔知道的更多，但到目前为止，她在所有公开场合都守口如瓶。

[b]通用信息理论(Universal Information Theory)：[/b]也被称为UnITy，它是一个试图诠释结构层，超链者的能力，共鸣域，和对其他矩阵现象的分类的统一理论。很多研究都是围绕着心智层展开的。

[b]偏差(Variance)：[/b]用于描述范式期望与狂奔者行动间差异的术语。这很糟糕，因为随着差异变大，整个主机结构层都会出来找你麻烦。

[b]广域网(WAN)：[/b]广阔区域网络. 用于描述设备可以从属于分布式框架式主机的地理区域(规模3到6)，大多是是彼此重叠的局域网。

[size=16pt][b]DEFINITIONS OF MATRIX TERMS[/b][/size]

[b]Base: [/b]This is an alternate term used for the basement of a framework host.

[b]Basement: [/b]Operating files, ownership structure, and archive of a framework host.

[b]Brick: [/b]Slang term for an optical stack mainframe.

[b]Cyberterminal: [/b]Desktop computer combining the functions of a cyberdeck and cyberjack. These units are not portable.

[b]Dataterm: [/b]Common home computer, home entertainment system, and communication device. These units are not portable.

[b]Deep Foundation: [/b]The Foundation layer of the Matrix itself, connecting all host Foundations.

[b]Foundation: [/b]Substructure of a Foundation host that runs the host’s operating code. This layer sits between the Resonance Realms and hosts.

[b]Foundation-Tech: [/b]The technology used to create and maintain Foundation hosts.

[b]Framework: [/b]A host generated by physical processors instead of a Foundation. These hosts consist of optical stack mainframes. Sometimes called legacy hosts.

[b]Mixed User Groups (MUGs): [/b]Acronym to describe most Matrix groups in existence.

[b]New Matrix: [/b]Shorthand for changes made to the Matrix in 2075 by Danielle de la Mar (not personally, but under her direction). At this point, it’s not really new, but surly old deckers always have a chip on their shoulder.

[b]New Matrix Protocols: [/b]See New Matrix.

[b]Noosphere: [/b]An energy field generated by information transmission and storage. This is based on the universal information theory.

[b]Nous-Particle Field Mechanics: [/b]The central component of universal information theory. This refers to how data and information are connected through hosts, the mind, and the Resonance Realms.

[b]LAN: [/b]Local area network. Refers to the geographic area where devices can be slaved to a small Framework host (Scale 1 or 2).

[b]Optical Stack Mainframe: [/b]The standard high-power computing device for generating hosts.

[b]PAN: [/b]Personal area network. This is all the devices that are linked with your main device, usually either a commlink or cyberjack.

[b]Paradigm: [/b]This is the term for the Foundation where the environment including all Foundation avatars and objects are forced to match some look and feel. This is similar to a host’s sculpting, except way more invasive.

[b]Quad, Cube, and Wall: [/b]These are pre-networked packages of 4, 16, or 64 optical stack mainframes.

[b]Resonance Realms: [/b]The generally unknown structures underpinning the Deep Foundation. Technomancers know about it. Go ask them.

[b]Scaffolding: [/b]The system that runs the sculpting of a host. This is controlled by the scaffolding control node in the Foundation.

[b]Sculpting: [/b]The programming that defines the user interface of a host. Like the paradigm, only it doesn’t force you to wear some silly outfit or suddenly transform into a car.

[b]Shell: [/b]The host envelope before any sculpting is performed. It’s a host, but just barely.

[b]Transfer Circuit: [/b]The code used by CC licensed corporations to create Foundation host shells. This somehow magically grows a host out of the Foundation. I am sure de la Mar knows more, but so far she’s being tight-lipped about it publicly.

[b]Universal Information Theory: [/b]Also known as UnITy, it is an attempt at a unified theory explaining Foundations, technomancer abilities, the Resonance Realms, and an assortment of other Matrix phenomena. Much of this research centers around the Noosphere.

[b]Variance: [/b]A term used for the difference between what a paradigm expects and the runners’ actions. This is bad because when the variance gets high, then the entire host Foundation is going to be out to get you.

[b]WAN: [/b]Wide area network. Used to describe the geographic area where devices can be slaved to a distributed Framework host (scale 3 to scale 6) and is basically a network of overlapping LANs.
